# Investigation Rules and Standards

## Evidence Hierarchy
1. **Primary source** — Document text directly from FBI vault PDFs (OCR output)
2. **FOIA release** — Officially released government documents
3. **Archival source** — University/museum collections, NARA records
4. **Secondary source** — Biographies (Seifer), journalism (MuckRock), academic analysis
5. **Tertiary** — History.com, Wikipedia, general reference

Always tag confidence levels (1-5) based on source quality. Never upgrade confidence without upgrading the source.

## The Core Investigative Lens
The FBI files are the government's file ON Tesla, not Tesla's files. Every document records institutional behavior, not Tesla's work. Read for what the institution did, feared, and hid — not for what Tesla invented.

## Redaction Analysis Principles
- What is redacted reveals what was considered sensitive
- Mid-sentence redactions identify the exact sensitive detail
- Signature-level redactions indicate the signatory is sensitive
- Process redactions (incident → [hidden] → resolution) indicate the process itself was sensitive
- The absence of a document that should exist (e.g., authorization memo) is itself a redaction

## Pattern Recognition Standards
Patterns must be:
- Named descriptively (e.g., "The 3-Day Disposal")
- Categorized: INSTITUTIONAL, HUMAN-NATURE, or SUPPRESSION
- Rated by strength (1-5)
- Connected to specific entities and evidence
- Compared to historical analogues where possible

## What To Watch For in OCR Text
Per LOG-002:
- Date anomalies (anything contradicting official timeline)
- Named third parties (unredacted names = leads)
- Document cross-references (file numbers outside the Tesla vault = separately FOIAble)
- Institutional language shifts (tone changes = different author or audience)
- Physical inventory descriptions (cross-referenceable against Belgrade)
- Geographic references (location = program/office/facility)
- Timeline compression (events faster than physically plausible)
